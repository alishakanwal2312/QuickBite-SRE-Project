# QuickBite – Online Food Delivery System

## Project Overview

QuickBite is an Online Food Delivery System developed as a semester project for the Software Requirements Engineering Lab. The system is designed to connect customers, restaurants, delivery riders, and administrators through a single platform to simplify the food ordering and delivery process.

The project focuses on requirement engineering activities including stakeholder analysis, requirements elicitation, requirement specification, UML modeling, and system documentation following the IEEE 830 Software Requirements Specification (SRS) standard.


## Project Objectives

- Provide a simple and user-friendly food ordering platform
- Reduce delays and mistakes in food ordering
- Improve communication between customers, restaurants, and delivery riders
- Provide secure payment options
- Enable real-time order tracking
- Help restaurants manage orders digitally
- Improve customer satisfaction

## Stakeholders

| Stakeholder   Role |
|------------|------|
| Customer | Places food orders and tracks deliveries |
| Restaurant Owner | Manages menu items and customer orders |
| Delivery Rider | Delivers food orders |
| System Administrator | Manages users and system activities |
| Payment Service Provider | Handles online payment processing |
| Customer Support Staff | Resolves customer complaints |

- User Registration

- Password Recovery
- Restaurant Browsing
- Food Search
- Menu Viewing
- Cart Management
- Order Placement
- Online Payment
- Cash on Delivery
- Order Notifications
- Order Acceptance/Rejection
- Order Status Updates
- Rider Assignment
- Delivery Tracking
- Order History
- Ratings and Reviews
- Admin Management


## Non-Functional Requirements

- Fast Response Time
- Security and Password Encryption
- Reliability
- Availability (24/7)
- Scalability
- Usability
- Maintainability
- Compatibility with Mobile and Web Platforms
